
# ملخص ميزات نظام الرواتب "حاسبني"

## 1. نظرة عامة

"حاسبني" هو نظام متكامل لإدارة شؤون الموظفين والرواتب، تم تصميمه ليكون فعالًا وذكيًا. يهدف النظام إلى أتمتة العمليات الروتينية، وتقليل الأخطاء البشرية، وتقديم رؤى تحليلية لدعم اتخاذ القرار. تم بناء النظام باستخدام تقنيات حديثة (Next.js, React, Firebase, Tailwind CSS) مما يجعله سريعًا وآمنًا وقابلاً للتطوير.

---

## 2. الوحدات الأساسية والميزات

### 2.1. لوحة المعلومات الرئيسية (Dashboard)
هي المركز التفاعلي الذي يعرض ملخصًا شاملاً لأهم البيانات والمؤشرات.
- **إحصائيات سريعة:** عرض عدد الموظفين النشطين، إجمالي الرواتب للشهر الحالي، وتكاليف العمل الإضافي.
- **نظام التنبيهات الذكية:** وحدة مدمجة تراقب البيانات بشكل استباقي وتصدر تنبيهات حيوية.
- **وصول سريع:** أزرار للانتقال السريع إلى الوحدات الرئيسية مثل إضافة موظف جديد أو معالجة الرواتب.

### 2.2. إدارة الموظفين (Employee Management)
نظام كامل لإدارة دورة حياة الموظف داخل المنشأة.
- **ملفات تعريف شاملة:** لكل موظف ملف شخصي يحتوي على:
    - المعلومات الشخصية والوظيفية.
    - تفاصيل العقد (تاريخ البدء، تاريخ الانتهاء، المسمى الوظيفي).
    - تفاصيل الراتب والبدلات (أساسي، سكن، مواصلات، إلخ).
    - السلف والعقوبات.
- **إدارة الحالات:** أرشفة الموظفين المنتهية خدماتهم بدلاً من حذفهم للحفاظ على سجلاتهم.
- **بحث وتصفية:** إمكانية البحث السريع عن الموظفين وتصفيتهم.

### 2.3. إدارة الحضور والانصراف (Attendance Tracking)
وحدة مرئية وتفاعلية لتتبع حضور الموظفين بدقة وسهولة.
- **جدول الحضور الشهري:** عرض شبكي يوضح حالة كل موظف لكل يوم في الشهر.
- **حالات مرمزة بالألوان:**
    - **أخضر:** حاضر
    - **أحمر:** غائب
    - **أصفر:** إجازة مرضية
    - **أزرق:** إجازة سنوية
    - **رمادي:** يوم عطلة (الجمعة)
- **التسجيل والتعديل السريع:** بالضغط على أي يوم، تظهر نافذة منبثقة تتيح:
    - تسجيل الحالة (حاضر، غائب، إجازة مرضية).
    - إدخال وقت الحضور والانصراف.
    - إضافة ملاحظات لليوم المحدد.
- **حساب الساعات:** يقوم النظام تلقائيًا بحساب ساعات العمل العادية والإضافية بناءً على أوقات الدخول والخروج.

### 2.4. معالجة الرواتب (Payroll Processing)
محرك حساب الرواتب الآلي الذي يضمن الدقة والسرعة.
- **حساب تلقائي:** يقوم النظام شهريًا بحساب صافي الراتب لكل موظف بناءً على:
    - الراتب الأساسي والبدلات.
    - خصومات الغياب (محسوبة من سجل الحضور).
    - أجر العمل الإضافي (محسوب من سجل الحضور).
    - السلف والجزاءات.
- **كشف رواتب تفصيلي:** إنشاء كشف راتب واضح لكل موظف يوضح جميع مكونات الراتب والخصومات.
- **سجل الرواتب:** الاحتفاظ بأرشيف لجميع كشوفات الرواتب التاريخية لكل موظف.
- **تأكيد الدفع:** إمكانية وضع علامة "مدفوع" على راتب الشهر لتسهيل المراجعة.

### 2.5. إدارة الإجازات (Leave Management)
أتمتة كاملة لعملية طلبات الإجازة والموافقة عليها.
- **تقديم الطلبات:** يمكن للموظفين تقديم طلبات الإجازة (سنوية أو مرضية) عبر النظام.
- **دورة الموافقة:** يمكن للمدراء مراجعة الطلبات والموافقة عليها أو رفضها.
- **التكامل التلقائي مع الحضور:** **عند الموافقة على طلب إجازة، يقوم النظام تلقائيًا بتحديث سجل الحضور للموظف للفترة المحددة، مما يلغي الحاجة للإدخال اليدوي.**

### 2.6. التسويات المالية (Financial Settlements)
وحدة متخصصة لحساب المستحقات النهائية للموظفين عند انتهاء خدمتهم.
- **حساب دقيق:** تقوم الوحدة بحساب مستحقات نهاية الخدمة، ورصيد الإجازات، وأي مستحقات أو خصومات أخرى.
- **إنهاء الخدمة:** تسجيل وتوثيق عملية التسوية بشكل كامل.

## 3. الميزات الذكية والتحليلية

### 3.1. نظام التنبيهات الذكية (Smart Alerts System)
نظام استباقي يراقب البيانات باستمرار لتنبيه الإدارة بالقضايا المحتملة.
- **مراقبة انتهاء العقود:** يصدر تنبيهات للعقود التي قاربت على الانتهاء (خلال 30، 15، أو 7 أيام).
- **مراقبة الغياب المفرط:** ينبه عند تجاوز الموظف لحد معين من الغياب خلال الشهر.
- **التحقق من سلامة البيانات:** يكتشف المشاكل المحتملة في بيانات الرواتب، مثل الأرقام السالبة أو الحقول الفارغة.
- **تصنيف الخطورة:** يتم تلوين التنبيهات (عالي، متوسط، منخفض) لتحديد الأولويات.

### 3.2. التحليل التنبؤي بالذكاء الاصطناعي (AI-Powered Predictive Analysis)
أداة لتحليل الاتجاهات المالية للموظف باستخدام الذكاء الاصطناعي.
- **تحليل الاتجاهات:** بناءً على بيانات الرواتب لآخر 6 أشهر، تقوم الأداة بتحليل التغيرات في (إجمالي الراتب، الخصومات، صافي الراتب).
- **رؤى باللغة الطبيعية:** يقدم التحليل ملخصًا نصيًا واضحًا، مثل: "يلاحظ وجود زيادة مستمرة في صافي الراتب خلال الأشهر الثلاثة الماضية نتيجة لزيادة ساعات العمل الإضافي."

## 4. البنية التقنية

- **الواجهة الأمامية:** React / Next.js
- **الواجهة الخلفية وقاعدة البيانات:** Firebase (Firestore, Functions)
- **التصميم:** Tailwind CSS
- **الحالة:** React Context & SWR for data fetching

هذا الملخص يوفر نظرة شاملة وعميقة على قدرات نظام "حاسبني"، مما يجعله إضافة قوية ومثالية للاندماج ضمن منظومة "وردة ERP".
